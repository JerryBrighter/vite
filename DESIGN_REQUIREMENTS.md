# CSV文件可视化分析工具 - 详细设计需求文档

## 1. 概述

### 1.1 项目简介

CSV文件可视化分析工具是一个基于浏览器的数据分析工具，支持上传、解析和可视化各种格式的数据文件，特别是CSV、DAT、TXT和Excel文件。该工具具有强大的编码检测和处理能力，能够正确处理GB2312、GBK和UTF-8等多种编码格式的文件。

### 1.2 核心定位

- **目标用户**：需要分析时序数据的工程师、数据分析师
- **核心价值**：提供便捷的数据可视化和分析能力
- **技术栈**：纯HTML/CSS/JavaScript + Bootstrap 5 + Chart.js + xlsx.js

## 2.7 文件预处理规范

### 2.7.1 文件格式识别规则

| 文件类型 | 识别方式 | 预处理策略 |
| :--- | :--- | :--- |
| **CSV文件** | 扩展名 `.csv` | 逗号分隔解析 |
| **TXT文件** | 扩展名 `.txt` | 自动检测分隔符类型 |
| **DAT文件** | 扩展名 `.dat` | 自动检测分隔符类型 |
| **Excel文件** | 扩展名 `.xlsx`/`.xls` | XLSX库解析 |

### 2.7.2 文本文件分隔符检测

#### 2.7.2.1 TAB分隔格式

**识别条件**：
- 文件内容包含TAB字符（`\t`）
- 数据行使用TAB分隔多列

**处理流程**：
```
读取文件内容
    │
    ▼
检测TAB字符存在
    │
    ▼
按TAB分割每行
    │
    ▼
处理标题行多余TAB的情况
    │
    ▼
生成二维数组
```

**特殊处理**：
- 如果标题行以TAB开头，且数据行比标题行少一列，则去掉标题行的第一个TAB
- 支持标题行包含冒号键值对格式（如 `温度:25`）

#### 2.7.2.2 空格分隔格式

**识别条件**：
- 第一列包含日期格式（年月日）
- 第二列包含时间格式（时分秒）
- 后续列为数值数据
- 使用空格分隔

**处理流程**：
```
读取文件内容
    │
    ▼
检测第一行是否符合日期 时间 数据1 数据2...格式
    │
    ▼
按空格分割每行
    │
    ▼
合并日期和时间列为完整日期时间
    │
    ▼
生成二维数组，标题为 时间、列1、列2...
```

**示例**：
```
输入：
2026-03-14 03:59:33 99.451 23.570
2026-03-14 03:59:34 98.234 24.123

输出：
[
  ['时间', '列1', '列2'],
  ['2026-03-14 03:59:33', '99.451', '23.570'],
  ['2026-03-14 03:59:34', '98.234', '24.123']
]
```

#### 2.7.2.3 冒号键值对格式

**识别条件**：
- 每行包含多个 `键:值` 对
- 使用TAB分隔不同的键值对

**处理流程**：
```
读取文件内容
    │
    ▼
检测是否包含冒号分隔的键值对
    │
    ▼
提取第一行的键作为标题
    │
    ▼
提取每行的值作为数据
    │
    ▼
生成二维数组
```

**示例**：
```
输入：
时间	温度:25	湿度:60
00:00:00	温度:26	湿度:58

输出：
[
  ['时间', '温度', '湿度'],
  ['00:00:00', '26', '58']
]
```

#### 2.7.2.4 逗号分隔格式（CSV）

**识别条件**：
- 文件内容包含逗号分隔符
- 符合标准CSV格式

**处理流程**：
```
读取文件内容
    │
    ▼
按换行分割行
    │
    ▼
解析CSV格式（处理引号包裹的内容）
    │
    ▼
生成二维数组
```

### 2.7.3 Excel文件预处理

#### 2.7.3.1 普通Excel格式

**处理流程**：
```
上传Excel文件
    │
    ▼
解析工作簿
    │
    ▼
显示Sheet选择器（多Sheet时）
    │
    ▼
用户选择Sheet
    │
    ▼
转换为二维数组
    │
    ▼
生成标准化表格
```

#### 2.7.3.2 特殊多Sheet格式

**识别条件**：
- 包含2个或以上Sheet
- 每个Sheet包含表头：`时间`、`名称`、`值`、`站点ID`、`设备ID`

**处理流程**：
```
上传Excel文件
    │
    ▼
检测是否为特殊多Sheet格式
    │
    ├─ 是
    │       │
    │       ▼
    │   遍历所有Sheet
    │       │
    │       ▼
    │   提取时间、名称、值列
    │       │
    │       ▼
    │   获取参数名称（第一行名称列值）
    │       │
    │       ▼
    │   时间对齐合并（使用Map按时间戳聚合）
    │       │
    │       ▼
    │   生成标准化大表
    │
    └─ 否
            │
            ▼
        显示Sheet选择器
```

**输出格式**：
| 时间 | 参数名_Sheet1 | 参数名_Sheet2 | ... |
|------|--------------|--------------|-----|
| 2026-05-07 18:11:40 | 值1 | 值2 | ... |
| 2026-05-07 18:11:41 | 值3 | 值4 | ... |

### 2.7.4 编码检测与处理

#### 2.7.4.1 编码自动检测

**检测顺序**：
1. **UTF-8 BOM** - 检测文件头 `0xEF 0xBB 0xBF`
2. **GB2312/GBK/GB18030** - 检测中文特征字符
3. **UTF-8** - 尝试UTF-8解码（fatal模式）
4. **默认** - UTF-8

**检测流程**：
```
读取文件二进制数据
    │
    ▼
检测UTF-8 BOM
    │
    ├─ 存在 ──► 使用UTF-8解码（忽略BOM）
    │
    └─ 不存在
            │
            ▼
        检测GB2312特征
            │
            ├─ 存在 ──► 使用GB18030解码
            │
            └─ 不存在
                    │
                    ▼
                尝试UTF-8解码（fatal模式）
                    │
                    ├─ 成功 ──► 使用UTF-8解码
                    │
                    └─ 失败 ──► 使用GB18030解码
```

#### 2.7.4.2 手动编码选择

**支持的编码**：
| 编码名称 | 说明 | 适用场景 |
| :--- | :--- | :--- |
| `auto` | 自动检测 | 默认选项 |
| `utf8` | UTF-8 | 通用编码 |
| `gb2312` | GB2312 | 简体中文 |
| `gbk` | GBK | 扩展中文 |
| `gb18030` | GB18030 | 全面中文支持 |

**处理流程**：
```
用户勾选"手动选择编码"
    │
    ▼
显示编码选择器
    │
    ▼
用户选择编码
    │
    ▼
应用选择的编码解码文件
    │
    ▼
显示解码预览
    │
    ▼
用户确认或修改编码
```

### 2.7.5 数据标准化流程

```
原始文件
    │
    ▼
格式识别（CSV/TXT/Excel/特殊多Sheet）
    │
    ▼
编码检测与解码
    │
    ▼
结构解析（分隔符识别/Sheet处理）
    │
    ▼
标题行处理
    │
    ├─ 已有标题行（第一列是"时间"）
    │       │
    │       ▼
    │   直接使用作为表头
    │
    └─ 无标题行
            │
            ▼
        检测前3行是否为标题
            │
            ├─ 是 ──► 合并多行标题
            │
            └─ 否 ──► 生成默认标题（时间、Column1、Column2...）
                    │
                    ▼
              数据行提取
                    │
                    ▼
              空列过滤
                    │
                    ▼
              零列过滤（可选）
                    │
                    ▼
              生成标准化二维数组
                    │
                    ▼
              保存原始数据 + 处理后数据
```

### 2.7.6 标题行检测规则

| 检测项 | 判断条件 | 结论 |
| :--- | :--- | :--- |
| **第一列是"时间"** | `data[0][0] === '时间'` | 已有标题行 |
| **第一列是时间格式** | 能解析为时间戳 | 无标题行 |
| **超过50%列是数字** | 数据行特征 | 无标题行 |
| **连续多行非数字** | 标题行特征 | 多行标题 |

### 2.7.7 数据清洗规则

| 清洗项 | 规则 | 处理方式 |
| :--- | :--- | :--- |
| **空行** | 整行数据为空 | 跳过该行 |
| **空列** | 所有行该列都为空 | 删除该列 |
| **全0列** | 所有行该列值为0 | 根据用户选择决定是否删除 |
| **空格清理** | 单元格前后有空格 | 去除首尾空格 |

## 4. 数据流程

### 4.1 文件上传流程

```
用户上传文件
    │
    ▼
检测文件类型
    │
    ├─ Excel文件 ──► 显示Sheet选择器 ──► 用户选择Sheet ──► 解析数据
    │
    └─ 其他文件 ──► 检测编码 ──► 解析数据
                        │
                        ├─ 自动检测成功 ──► 解析数据
                        │
                        └─ 手动选择编码 ──► 解析数据
```

### 4.2 数据处理流程

```
用户上传文件
    │
    ▼
文档格式识别
    │
    ├─ CSV格式检测
    ├─ TAB分隔检测
    ├─ 空格分隔检测
    └─ 冒号键值对检测
            │
            ▼
    统一为标准化表格
            │
            ▼
    ┌────────────────┐     ┌────────────────┐
    │  保存原始数据  │     │  创建处理后数据 │
    │ (用于表格显示) │     │ (用于图表/导出) │
    └────────────────┘     └────────────────┘
            │                      │
            ▼                      ▼
    显示原始格式表格        应用零列过滤、时间规范化等处理
```

### 4.3 日期识别流程

```
数据文件加载完成
    │
    ▼
检测时间列是否包含日期
    │
    ├─ 包含日期 ──► 直接使用数据中的日期
    │
    └─ 不包含日期 ──► 尝试从文件名解析日期
                            │
                            ├─ 解析成功 ──► 使用文件名日期
                            │
                            └─ 解析失败 ──► 使用文件修改时间
```

### 4.4 图表绘制流程

```
用户点击绘制按钮
    │
    ▼
获取X轴和Y轴选择
    │
    ▼
验证选择有效性
    │
    ├─ 无效 ──► 提示用户选择
    │
    └─ 有效 ──► 处理大数据抽样
                        │
                        ▼
                  规范化时间格式
                        │
                        ▼
                  创建Canvas元素
                        │
                        ▼
                  配置Chart.js图表
                        │
                        ▼
                  渲染图表
                        │
                        ▼
                  添加图例交互
```
