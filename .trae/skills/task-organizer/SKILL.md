# 任务组织SKILL

## 概述

本SKILL负责开发任务的组织和管理，确保开发工作有序进行。核心职责是将用户需求转化为结构化、可执行的任务清单。

## 工作流程

### 1. 需求分析阶段

**输入**：用户需求描述

**输出**：需求分析报告

#### 1.1 需求理解
- 深入理解用户需求
- 识别需求中的核心功能点
- 判断需求的优先级（高/中/低）
- 评估需求的复杂度和风险

#### 1.2 需求文档化
- 创建需求文档（参考dev-standard SKILL）
- 文档位置：`.trae/documents/<feature-name>-requirement.md`
- 包含：需求概述、功能说明、技术方案、影响范围、验收标准

### 2. 任务拆分阶段

**输入**：需求分析报告

**输出**：结构化任务清单

#### 2.1 任务分解
- 将需求分解为独立的任务单元
- 每个任务应遵循"单一职责"原则
- 任务粒度适中（建议每个任务1-4小时工作量）

#### 2.2 任务优先级排序
- 使用优先级矩阵进行排序：
  - P0：紧急且重要（必须立即处理）
  - P1：重要但不紧急（按计划处理）
  - P2：紧急但不重要（尽快处理）
  - P3：不紧急不重要（空闲时处理）

#### 2.3 任务依赖分析
- 识别任务之间的依赖关系
- 确定任务执行顺序
- 标记并行执行的任务

### 3. 任务清单输出

**格式**：JSON结构或Markdown表格

#### 3.1 任务清单结构
```json
{
  "project": "项目名称",
  "version": "目标版本",
  "createdAt": "创建时间",
  "tasks": [
    {
      "id": "T001",
      "title": "任务标题",
      "description": "任务详细描述",
      "priority": "P0",
      "dependencies": ["T002"],
      "estimatedHours": 2,
      "status": "pending",
      "assignee": "code-writer",
      "requirements": "需求文档路径",
      "acceptanceCriteria": "验收标准"
    }
  ]
}
```

#### 3.2 Markdown表格格式
| 任务ID | 标题 | 优先级 | 预估工时 | 依赖任务 | 状态 |
|--------|------|--------|----------|----------|------|
| T001 | 任务1 | P0 | 2h | - | pending |
| T002 | 任务2 | P1 | 4h | T001 | pending |

### 4. 进度跟踪阶段

**输入**：任务执行状态

**输出**：进度报告

#### 4.1 状态更新
- 定期更新任务状态：pending → in_progress → completed → blocked
- 记录任务开始时间和完成时间
- 标记阻塞原因和解决时间

#### 4.2 进度报告
- 生成每日/每周进度报告
- 包含：完成率、阻塞问题、风险评估、下周计划

## 输出产物

| 产物 | 格式 | 用途 |
|------|------|------|
| 需求分析报告 | Markdown | 记录需求理解和技术方案 |
| 任务清单 | JSON/Markdown | 指导开发工作 |
| 进度报告 | Markdown | 跟踪开发进度 |

## 与其他SKILL的协作

### code-writer
- **输入**：任务清单
- **输出**：完成的代码
- **交互**：任务组织SKILL输出任务清单，code-writer按清单执行

### code-reviewer
- **输入**：完成的代码和任务清单
- **输出**：审核报告
- **交互**：code-reviewer根据任务清单审核代码是否符合需求

### dev-standard
- **引用**：作为基础规范，确保所有操作符合开发标准
- **备份**：任务开始前调用dev-standard的备份流程
- **测试**：任务完成后调用dev-standard的测试流程

## 最佳实践

1. **清晰的需求描述**：确保每个任务都有明确的目标和验收标准
2. **合理的任务粒度**：避免任务过大或过小，建议1-4小时每个任务
3. **及时的状态更新**：保持任务状态最新，便于团队协作
4. **风险预判**：在任务清单中识别潜在风险并制定应对方案
5. **文档同步**：任务变更时同步更新相关文档

## 工具集成

- **TodoWrite**：用于任务管理和进度跟踪
- **AskUserQuestion**：用于需求确认和优先级确认
- **dev-standard**：用于代码备份和测试验证

## 附录

### 优先级定义
| 优先级 | 定义 | 处理策略 |
|--------|------|----------|
| P0 | 紧急且重要 | 立即处理，优先于其他任务 |
| P1 | 重要但不紧急 | 按计划处理，纳入当前迭代 |
| P2 | 紧急但不重要 | 尽快处理，可以委托他人 |
| P3 | 不紧急不重要 | 空闲时处理，或延后到下一个迭代 |

### 任务状态定义
| 状态 | 定义 |
|------|------|
| pending | 待开始 |
| in_progress | 进行中 |
| completed | 已完成 |
| blocked | 阻塞中 |
| cancelled | 已取消 |
